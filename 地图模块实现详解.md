# 地图模块实现详解

## 📍 一、地图模块整体架构

该项目使用**高德地图**作为地图服务提供商，采用前后端分离的架构实现地图功能。整体架构如下：

```
前端（Vue 3） ←→ 后端（Spring Boot） ←→ 高德地图API
     ↓                    ↓                    ↓
AmapSelector.vue    AmapController      高德地图服务
     ↓                    ↓
燃气用户管理页面    AmapService
                         ↓
                    AmapConfig（配置）
```

---

## 🎨 二、前端实现详解

### 1. 核心组件：AmapSelector.vue

**文件路径**：`gas-vue/src/components/AmapSelector.vue`

这是一个功能完整的地图选择器组件，主要功能包括：

#### 1.1 组件结构

- **地图容器**：显示高德地图
- **搜索框**：支持地址搜索和自动提示
- **POI列表**：显示附近兴趣点
- **当前地址显示**：实时显示选中的地址

#### 1.2 核心功能实现

##### ① 地图初始化流程

```
initMap() → loadAmapConfig() → loadAmapScript() → createMap()
```

- **步骤1**：从后端获取地图配置（密钥等）
- **步骤2**：动态加载高德地图JS SDK
- **步骤3**：创建地图实例和相关插件（地理编码、地点搜索）

##### ② 地图配置获取

**代码位置**：第120-129行

```javascript
loadAmapConfig() {
    return new Promise((resolve, reject) => {
        this.$http('/amap/config', 'GET', {}, true, (resp) => {
            if (resp && (resp.code === 0 || resp.code === 200)) {
                resolve(resp.result);
            }
        });
    });
}
```

##### ③ 地图SDK动态加载

**代码位置**：第131-152行

- 设置安全密钥：`window._AMapSecurityConfig`
- 动态创建script标签加载高德地图API
- 使用回调函数确保SDK加载完成

##### ④ 地图交互功能

**点击地图选择位置**（第175-177行）：

```javascript
this.map.on('click', (e) => {
    this.onMapClick(e.lnglat);
});
```

**地址搜索**（第311-334行）：

- 支持关键词搜索
- 显示搜索结果列表
- 点击结果定位到地图

**附近POI搜索**（第268-301行）：

- 根据点击位置搜索附近1000米内的兴趣点
- 显示POI列表，包含名称、地址、距离

**逆地理编码**（第257-266行）：

- 将经纬度坐标转换为详细地址
- 实时更新当前地址显示

#### 1.3 数据流

```
用户操作 → 地图交互 → 获取位置信息 → 更新组件状态 → 触发select事件
```

## ⚙️ 三、后端实现详解

### 1. 配置类：AmapConfig.java

**文件路径**：`gas-api/src/main/java/com/example/gas/api/config/AmapConfig.java`

使用Spring Boot的配置属性绑定：

```java
@Data
@Component
@ConfigurationProperties(prefix = "amap")
public class AmapConfig {
    private String webKey;           // Web端API密钥
    private String webServiceKey;    // Web服务API密钥
    private String securityJsCode;   // JS安全密钥
    private String privateKey;       // 数字签名私钥
}
```

### 2. 服务层：AmapService.java

**文件路径**：`gas-api/src/main/java/com/example/gas/api/service/AmapService.java`

#### 2.1 核心API封装

该服务封装了高德地图的多个REST API：

| API方法 | 功能 | 高德API端点 |
|---------|------|------------|
| `geocode()` | 地址转经纬度 | `/geocode/geo` |
| `regeocode()` | 经纬度转地址 | `/geocode/regeo` |
| `inputTips()` | 输入提示 | `/assistant/inputtips` |
| `searchPoi()` | 关键词搜索POI | `/place/text` |
| `searchNearby()` | 周边搜索POI | `/place/around` |
| `getPoiDetail()` | POI详情 | `/place/detail` |

#### 2.2 请求签名机制

**代码位置**：第118-158行

为了提高安全性，实现了数字签名机制：

```java
private String generateSignature(Map<String, Object> params) {
    // 1. 参数按key排序
    // 2. 拼接成字符串：key1=value1&key2=value2...privateKey
    // 3. MD5加密生成签名
}
```

**签名流程**：

1. 将所有请求参数按字母顺序排序
2. 拼接成 `key=value&key=value` 格式
3. 在末尾追加私钥
4. 对整个字符串进行MD5加密
5. 将签名添加到请求参数中

#### 2.3 基础参数构建

**代码位置**：第108-116行

```java
private Map<String, Object> buildBaseParams() {
    Map<String, Object> params = new HashMap<>();
    params.put("key", amapConfig.getWebServiceKey());
    params.put("output", "json");
    return params;
}
```

### 3. 控制器层：AmapController.java

**文件路径**：`gas-api/src/main/java/com/example/gas/api/controller/AmapController.java`

提供RESTful API接口，所有接口都需要登录认证（`@SaCheckLogin`）：

| 接口路径 | HTTP方法 | 功能 | 参数 |
|---------|---------|------|------|
| `/amap/config` | GET | 获取前端配置 | 无 |
| `/amap/geocode` | GET | 地址转坐标 | address, city |
| `/amap/regeocode` | GET | 坐标转地址 | location, extensions |
| `/amap/inputTips` | GET | 输入提示 | keywords, city, cityLimit |
| `/amap/searchPoi` | GET | POI搜索 | keywords, city, cityLimit, offset, page |
| `/amap/searchNearby` | GET | 周边搜索 | location, radius, offset, page |
| `/amap/poiDetail` | GET | POI详情 | id |

## 🔄 五、完整调用流程

### 用户选择地址的完整流程

```
1. 用户点击"地图选择"按钮
   ↓
2. 打开AmapSelector对话框
   ↓
3. 组件初始化：
   - 调用 /amap/config 获取地图配置
   - 加载高德地图SDK
   - 创建地图实例
   ↓
4. 用户交互：
   - 点击地图 → 触发逆地理编码 → 显示地址
   - 搜索地址 → 调用 /amap/searchPoi → 显示结果
   - 点击附近POI → 调用 /amap/searchNearby → 显示列表
   ↓
5. 用户确认选择
   ↓
6. 触发select事件，传递地址和位置信息
   ↓
7. 父组件接收数据，更新表单
   ↓
8. 提交表单，保存到数据库
```

## 🌟 六、技术亮点

### 1. 安全性设计

- 使用数字签名机制防止API滥用
- 前后端密钥分离（Web端和服务端使用不同密钥）
- JS安全密钥防止前端API被盗用

### 2. 性能优化

- 动态加载地图SDK，减少首屏加载时间
- 使用防抖处理搜索输入（300ms延迟）
- 地图实例在对话框关闭时销毁，释放资源

### 3. 用户体验

- 提供多种地址选择方式（地图点击、搜索、附近POI）
- 实时显示当前选中地址
- 支持行政区域联动定位

### 4. 代码规范

- 前后端分离，职责清晰
- 使用Promise处理异步操作
- 统一的错误处理机制

## 🎯 八、总结

该项目的地图模块采用了**前后端分离**的架构设计，前端使用Vue 3组件化开发，后端使用Spring Boot提供服务，通过高德地图API实现地图功能。整个模块设计合理，代码结构清晰，安全性高，用户体验良好。

地图功能主要用于燃气用户的地址选择，实现了地址搜索、地图点击定位、附近POI查询等核心功能，是一个典型的企业级地图应用实现案例。
