# 巡检人员登录注册优化建议

> 分析时间：2026-04-20
> 分析对象：patient-wx 小程序项目
> 分析范围：登录、注册、实名登记流程

---

## 一、当前登录注册流程概述

### 1.1 现有流程

```
用户首次使用：
点击"注册/登录" → 微信授权 → 自动创建账号 → 登录成功
                                    ↓
                              显示"未绑定手机（需实名登记）"
                                    ↓
点击"实名登记" → 填写个人信息 → 根据手机号匹配巡检人员
```

### 1.2 数据库表关系

```
PATIENT_USER (小程序用户表)
    ↓ user_id
PATIENT_USER_INFO_CARD (用户信息卡表)
    ↓ worker_id (通过手机号匹配)
gas_worker (巡检人员表)
```

### 1.3 关键代码位置

| 功能 | 前端文件 | 后端文件 |
|------|---------|----------|
| 登录注册 | `patient-wx/pages/mine/mine.vue` | `patient-wx-api/.../UserServiceImpl.java` |
| 实名登记 | `patient-wx/user/fill_user_info/fill_user_info.vue` | `patient-wx-api/.../UserInfoCardServiceImpl.java` |
| 巡检员信息 | `patient-wx/user/gas_worker_info/gas_worker_info.vue` | `patient-wx-api/.../GasWorkerServiceImpl.java` |

## 二、从用户角度发现的问题

### 问题1：登录入口不够直观

**现状：**
- 用户看到的是"注册 / 登陆"两个词合并在一起
- 用户不清楚点击后会发生什么

**用户心理：**
> "我是要注册还是登录？这两个有什么区别？"

**优化建议：**
```
方案A：改为"微信一键登录"
方案B：改为"点击授权登录"
方案C：显示图标 + 文案，如 [微信图标] "使用微信登录"
```

**涉及文件：**
- `patient-wx/pages/mine/mine.vue` 第6-8行

---

### 问题2：实名登记入口隐藏较深

**现状：**
- 用户登录后看到"未绑定手机（需实名登记）"
- 需要点击"实名登记"才能填写信息
- 巡检人员可能不知道需要这一步才能关联身份

**用户心理：**
> "我登录了，为什么看不到我的巡检任务？我的信息在哪里？"

**优化建议：**
```
方案A：登录后自动检测是否已实名
       - 未实名 → 弹窗提示引导用户完成实名登记
       - 已实名 → 直接显示巡检人员信息

方案B：登录后首次进入"我的信息"页面时
       - 自动跳转到实名登记页面

方案C：在首页增加醒目的"完善信息"提示卡片
```

**涉及文件：**
- `patient-wx/pages/mine/mine.vue` - 登录成功后的回调处理
- `patient-wx/pages/index/index.vue` - 首页增加提示卡片

---

### 问题3：手机号匹配巡检人员缺乏反馈

**现状：**
- 用户填写手机号后提交
- 系统后台自动匹配巡检人员
- 用户不知道是否匹配成功

**用户心理：**
> "我填完信息了，系统知道我是巡检员了吗？"

**优化建议：**
```
方案A：实名登记成功后显示匹配结果
       - 匹配成功 → "已关联巡检员身份：张三"
       - 未匹配 → "暂未关联巡检员身份，请联系管理员"

方案B：在"我的信息"页面增加身份标识
       - 显示"巡检员"徽章或标签
```

**涉及文件：**
- `patient-wx/user/fill_user_info/fill_user_info.vue` - 提交成功后的处理
- `patient-wx-api/.../UserInfoCardController.java` - 返回匹配结果

---

### 问题4：微信授权流程可能失败

**现状：**
- 先调用 `uni.login()` 获取 code
- 再调用 `uni.getUserProfile()` 获取用户信息
- 如果用户拒绝授权，没有明确的错误处理

**用户心理：**
> "我点了取消，然后呢？什么都没发生？"

**优化建议：**
```
方案A：增加授权失败的友好提示
       "您拒绝了授权，无法完成登录，请重新点击授权"

方案B：提供备选方案
       "如果授权失败，您可以联系管理员手动添加"
```

**涉及文件：**
- `patient-wx/pages/mine/mine.vue` - `loginOrRegister` 方法增加 fail 回调

---

### 问题5：登录状态不明确

**现状：**
- 登录前后页面变化较小
- 仅通过文字变化区分状态

**用户心理：**
> "我现在登录了吗？怎么确认？"

**优化建议：**
```
方案A：登录后显示用户头像和昵称（已有）
方案B：增加"已登录"状态标识
方案C：未登录时显示灰色头像占位符，登录后显示真实头像
```

**涉及文件：**
- `patient-wx/pages/mine/mine.vue` - 头像和状态显示逻辑

---

### 问题6：退出登录功能缺失

**现状：**
- 代码中未看到退出登录的功能入口
- 用户无法主动退出账号

**用户心理：**
> "我想换个账号登录，怎么退出？"

**优化建议：**
```
方案A：在"我的"页面增加"退出登录"按钮
方案B：长按头像弹出退出选项
方案C：在设置页面提供退出功能
```

**涉及文件：**
- `patient-wx/pages/mine/mine.vue` - 增加退出登录按钮和方法
- 后端需要增加 `/user/logout` 接口

---

### 问题7：Token过期处理不够友好

**现状：**
- Token过期后返回401错误
- 显示"请登录小程序"提示
- 2秒后跳转到"我的"页面

**用户心理：**
> "我刚才不是登录了吗？怎么又要登录？"

**优化建议：**
```
方案A：增加Token自动刷新机制
方案B：提示更友好："登录已过期，请重新授权"
方案C：自动弹出授权弹窗，无需用户手动点击
```

**涉及文件：**
- `patient-wx/main.js` - ajax 封装函数中的 401 处理

## 三、优化建议汇总表

| 问题 | 优先级 | 优化方案 | 预期效果 | 涉及文件 |
|------|--------|----------|----------|----------|
| 登录入口不直观 | 高 | 改为"微信一键登录" | 降低用户困惑 | mine.vue |

## 四、核心优化方案详细设计

### 4.1 方案一：登录后自动检测并引导

**流程设计：**
```
用户登录成功
    ↓
检测是否已实名登记
    ├── 未实名 → 弹窗："检测到您尚未实名登记，是否立即完善？"
    │              [稍后再说] [立即登记]
    └── 已实名 → 检测是否关联巡检员
                   ├── 已关联 → 显示"欢迎回来，巡检员张三"
                   └── 未关联 → 显示"请联系管理员添加巡检员身份"
```

**前端实现思路：**
```javascript
// mine.vue - loginOrRegister 方法中
loginOrRegister: function() {
    // ... 现有登录逻辑
    that.ajax(that.api.loginOrRegister, 'POST', data, function(resp) {
        // 登录成功后，检测是否需要引导
        that.ajax(that.api.searchUserInfoCard, 'GET', {}, function(resp2) {
            let result = resp2.data.result;
            if (!result || Object.keys(result).length === 0) {
                // 未实名，弹出引导
                uni.showModal({
                    title: '提示',
                    content: '检测到您尚未实名登记，是否立即完善？',
                    confirmText: '立即登记',
                    cancelText: '稍后再说',
                    success: function(modalResp) {
                        if (modalResp.confirm) {
                            uni.navigateTo({
                                url: '/user/fill_user_info/fill_user_info'
                            });
                        }
                    }
                });
            }
        }, false);
    });
}
```

**后端接口调整：**
```java
// UserInfoCardController.java - 增加返回匹配结果
@PostMapping("/insert")
public R insert(@RequestBody @Valid InsertUserInfoCardForm form) {
    // ... 现有逻辑
    UserInfoCardEntity entity = new UserInfoCardEntity();
    // ...
    userInfoCardService.insert(entity);
    
    // 返回匹配结果
    R r = R.ok("登记成功");
    if (entity.getWorkerId() != null) {
        GasWorkerEntity worker = gasWorkerDao.searchById(entity.getWorkerId());
        if (worker != null) {
            r.put("workerName", worker.getName());
            r.put("matched", true);
        }
    } else {
        r.put("matched", false);
    }
    return r;
}
```

### 4.3 方案三：增加身份识别反馈

**UI设计：**
```
实名登记提交成功后：

┌─────────────────────────────────────┐
│            ✓ 登记成功               │
├─────────────────────────────────────┤
│  已关联身份：巡检员                  │
│  姓名：张三                          │
│  所属：巡检一组                      │
│                                     │
│  [查看我的任务]    [返回首页]        │
└─────────────────────────────────────┘
```

**前端实现思路：**
```javascript
// fill_user_info.vue - submitHandle 方法
submitHandle: function() {
    let that = this;
    uni.showModal({
        title: '提示消息',
        content: '确定提交？',
        success: function(resp) {
            if (resp.confirm) {
                that.ajax(that.api.insertUserInfoCard, 'POST', that.dataForm, function(resp) {
                    let data = resp.data;
                    that.showSuccessModal = true;
                    
                    if (data.matched) {
                        that.matched = true;
                        that.workerName = data.workerName;
                    } else {
                        that.matched = false;
                    }
                });
            }
        }
    });
}
```

## 七、注意事项

1. **向后兼容**：修改后端接口时，确保返回字段向后兼容，不影响现有功能
2. **测试覆盖**：修改后需要测试以下场景：
   - 新用户首次登录
   - 已注册用户登录
   - 实名登记匹配成功
   - 实名登记匹配失败
   - 授权取消/失败
   - Token过期处理

3. **数据一致性**：确保手机号匹配逻辑的准确性，避免误匹配
