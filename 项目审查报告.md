## 四、问题详细清单

### 4.1 P0级问题 - 紧急修复

#### 问题1：敏感信息明文存储

**严重程度**：🔴 严重

**影响范围**：gas-api、patient-wx-api

**问题描述**：
`application.yml` 中明文存储了所有敏感配置信息，包括：
- 数据库密码：`123456`
- Redis密码：`abc123456`
- 微信AppSecret：`b3c3d69cbc62b63cb9ba95b0e64d87dd`
- 腾讯TRTC SecretKey
- MinIO access-key/secret-key
- 高德地图 Private Key

**文件位置**：
- `gas-api/src/main/resources/application.yml` 第18、40、84-89行
- `patient-wx-api/src/main/resources/application.yml` 第19、39、82-99行

**风险**：
- 配置文件被提交到代码仓库后，敏感信息泄露
- 攻击者可直接获取数据库、云服务访问权限

**修复建议**：
1. 使用环境变量替代硬编码配置
2. 使用配置中心（如Nacos、Apollo）管理敏感配置
3. 使用Jasypt等工具加密配置文件

**影响范围**：patient-wx-api

| POST /hazard_info/wx/uploadRectificationVideo | **可上传任意文件** |

**修复建议**：

#### 问题3：路径遍历风险

**严重程度**：🔴 严重

**影响范围**：patient-wx-api

**问题描述**：
`ImageProxyController` 的 `path` 参数直接传入MinIO获取对象，未对 `../` 等路径遍历字符进行过滤。

**文件位置**：
- `patient-wx-api/src/main/java/com/example/gas/patient/wx/api/controller/ImageProxyController.java` 第31行

**风险**：
攻击者可通过构造 `path=../../../other-bucket/sensitive-file` 访问非预期文件

**修复建议**：
1. 对 `path` 参数进行严格校验，过滤 `../` 等危险字符
2. 使用白名单机制限制可访问的路径

#### 问题5：onLoad生命周期写在methods内部

**严重程度**：🔴 严重（功能性Bug）

**影响范围**：patient-wx

**问题描述**：
以下4个页面的 `onLoad` 方法写在了 `methods` 对象内部，而不是与 `methods` 同级的生命周期钩子位置，导致页面加载时不会执行初始化逻辑。

**受影响文件**：
- `patient-wx/registration/worker_inspection/worker_inspection.vue`
- `patient-wx/hazard_rectify/hazard_rectify_list/hazard_rectify_list.vue`
- `patient-wx/hazard_rectify/hazard_rectify_detail/hazard_rectify_detail.vue`
- `patient-wx/hazard_rectify/hazard_rectify_submit/hazard_rectify_submit.vue`

**错误示例**：
```javascript
export default {
    methods: {
        onLoad(option) {  // ❌ 错误位置
            // ...
        },
        otherMethod() {}
    }
}
```

**正确写法**：
```javascript
export default {
    onLoad(option) {  // ✅ 正确位置
        // ...
    },
    methods: {
        otherMethod() {}
    }
}
```

**修复建议**：
将 `onLoad` 从 `methods` 移到与 `methods` 同级的位置

---

#### 问题6：登录时序竞争

**严重程度**：🔴 严重

**影响范围**：patient-wx

**问题描述**：
`mine.vue` 中 `uni.login` 和 `uni.getUserProfile` 并行调用，存在时序竞争：`code` 可能在 `getUserProfile` 回调时还未赋值。

**文件位置**：
- `patient-wx/pages/mine/mine/mine.vue` 第82-131行

**错误代码**：
```javascript
let code = null;
uni.login({
    success: function(resp) {
        code = resp.code;  // 异步赋值
    }
});
uni.getUserProfile({
    success: function(resp) {
        // 此时 code 可能为 null
        that.ajax(that.api.loginOrRegister, 'POST', {
            code: code,  // ❌ 可能为 null
            nickname: resp.userInfo.nickName,
            photo: resp.userInfo.avatarUrl,
            sex: resp.userInfo.gender === 1 ? '男' : '女'
        }, ...)
    }
});
```

**修复建议**：
先获取 `code`，再调用 `getUserProfile`
